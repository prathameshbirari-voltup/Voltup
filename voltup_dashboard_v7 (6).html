Data & KPI Logic
Paybase — Active customers with ≥1 recharge in last 30 days. Target: 1,280 · Achieved: 55%
Swap Revenue — Total revenue from battery swap transactions. ₹13.4L · +6% MoM
Station Utilization — Avg swaps per station per day vs capacity threshold (3.5)
Overdue Recovery — Bikes not returned in <15 days. 162 overdue · 68% recovery rate
Cluster Split Logic — Noida City total = City Center (56%) + Noida Extension (44%)
Data Source — Live operational DB, refreshed every 15 minutes via secure API sync

How It Works

  1. App fetches real-time data from multiple systems
  2. KPIs calculated as per defined business formulas
  3. Compare target vs actual achievement
  4. Detect issues and trigger smart alerts
  5. Show trends, highlights and cluster rankings
  6. ASH takes action and submits market intel
  7. All data timestamped for monthly reports
  8. Helps in decision making and performance improvement

Data Sources

  • Customer and Billing System
  • Swap System (Swaps, Revenue, ARPU)
  • Fleet System (Stations, Bikes)
  • Station System (Slots, Batteries)
  • Recharge System (Overdue)
  • Operations / Asset Management
  • CRM / Cluster Master data
  • Manual Input (Market Intel)

Calculation Logic

  • Active Base = Active + Overdue less than 72H
  • Revenue Ach = (Actual / Target) x 100
  • Station Utz = Total Swaps / Active Slots
  • Bike Pickup = Picked / Total Overdue x 100
  • Recharge OT = On-time / Total x 100
  • Cluster Health: Green above 75, Yellow 50-75, Red below 50
